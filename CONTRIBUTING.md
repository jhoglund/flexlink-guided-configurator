# Contributing

## Branching
- main: stable
- feature/*: new features
- fix/*: bug fixes

## Commit messages
- Conventional style: `feat: ...`, `fix: ...`, `docs: ...`, `chore: ...`

## PR checklist
- [ ] Tests pass locally
- [ ] No secrets in code/docs
- [ ] Follows ERB/CSS style guide
- [ ] Updates docs if needed

## Setup
- `cp env.example .env`
- `./start.sh`
- `git-all "feat: ..."`
